## Coding Standards

### File Naming Convention
- Use PascalCase: `TwoSum.java`, `ContainsDuplicate.java`
- Include problem number: `LC001_TwoSum.java`

### Solution Template
```java

 *

 * Space Complexity: O(?)
 * 
 * Approach:
 * - Brief explanation of your solution
 */
class Solution {

    }
}
```

### Best Practices
1. **Comment Your Code**: Explain the "why", not just the "what"
2. **Analyze Complexity**: Always document time and space complexity
3. **Multiple Approaches**: If you find a better solution later, keep both with comments
4. **Test Cases**: Include edge cases in comments

## Tips for Success

### Daily Routine
1. **Warm-up** (10 min): Review previous solutions
2. **New Problem** (30-45 min): Attempt without hints
3. **Learn** (15 min): If stuck, watch explanation
4. **Implement** (20 min): Code the solution
5. **Reflect** (5 min): Document learnings

### When You're Stuck
1. Draw examples on paper
2. Start with brute force, then optimize
3. Think about what data structure could help
4. Check if you've seen a similar pattern before
5. Watch the NeetCode video explanation

### Before Interviews
- Review all problems you've solved (especially mistakes)
- Practice explaining your approach out loud
- Time yourself on similar problems
- Focus on patterns, not memorization
